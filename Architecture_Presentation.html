2. RAG Pipeline: Document Ingestion Flow

Illustrates the process when a syllabus or textbook PDF is uploaded. Highlights the use of the child process for extraction, chunking, embedding, and Pinecone storage.

sequenceDiagram autonumber participant User as React Frontend participant API as RAG Controller participant Worker as PDF Worker (Child Process) participant Gemini as Gemini Embedding Model participant Pinecone as Pinecone Vector DB participant MongoDB as MongoDB (Metadata) User->>API: Upload PDF Textbook API->>MongoDB: Create "Processing" Record API->>Worker: Dispatch PDF Extraction Task Worker-->>API: Return Extracted Text JSON API->>API: Clean & Chunk Text (800 chars, overlaps) loop For each batch of text chunks API->>Gemini: Request Embeddings (text-embedding-004) Gemini-->>API: Return Vector Array (768-dim) API->>Pinecone: Upsert Vectors (with Metadata) end API->>MongoDB: Update Status to "Ready" API-->>User: Return Success (Total Chunks, Pages)
